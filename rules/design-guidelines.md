# 设计规则

本文档定义通用的设计基础规则，适用于各种类型的原型开发。

## 🎨 设计原则

### 1. 一致性
- 同类元素使用统一的视觉样式
- 相同操作产生相同结果
- 术语和命名保持一致

### 2. 清晰性
- 信息层次分明，重要内容突出
- 避免不必要的装饰
- 文字易读，对比度充足

### 3. 可用性
- 功能和操作易于理解
- 提供及时的操作反馈
- 支持常见的交互模式

### 4. 可访问性
- 使用语义化 HTML 标签
- 支持键盘导航
- 提供必要的辅助说明

## 🎯 组件设计规范

### 元素（Element）
可复用的独立组件：
- **职责单一**：每个元素只负责一个明确的功能
- **高度可配置**：提供足够的配置项支持不同场景
- **独立性强**：不依赖特定页面结构或全局状态

### 页面（Page）
完整的业务场景：
- **业务完整**：处理加载、成功、失败、空数据等所有状态
- **布局合理**：信息层次清晰，重要内容优先显示
- **响应式**：适配不同屏幕尺寸，关键功能在小屏幕可用

## 🎨 视觉设计

### 色彩
- **主色**：用于主要操作和重点信息
- **辅助色**：用于状态反馈（成功、警告、错误、信息）
- **中性色**：用于文字、边框、背景
- 使用 CSS 变量支持主题定制

### 字体
- 使用系统字体栈确保兼容性
- 建立清晰的字号层级（标题、正文、辅助文字）
- 合理使用字重区分重要性

### 间距
- 使用统一的间距基准（如 4px、8px 的倍数）
- 保持元素间距的一致性
- 根据内容重要性调整间距大小

### 圆角与阴影
- 统一圆角风格
- 使用阴影表现层级关系
- 避免过度装饰

## 🖱️ 交互设计

### 按钮
- **状态反馈**：默认、悬停、激活、禁用状态
- **加载状态**：异步操作时显示加载提示
- **视觉反馈**：使用过渡动画提升体验

### 表单
- **输入状态**：默认、聚焦、错误、禁用状态
- **即时验证**：输入时提供实时反馈
- **错误提示**：清晰说明错误原因

### 列表
- **悬停效果**：鼠标悬停时高亮显示
- **选中状态**：明确标识选中项
- **空状态**：友好的空数据提示和引导

## 📱 响应式设计
- **移动优先**：从小屏幕开始设计，逐步增强
- **断点设置**：根据内容需求设置合理断点
- **灵活布局**：使用弹性布局和网格系统
- **关键功能**：确保核心功能在所有设备可用

## ♿ 可访问性
- **语义化**：使用正确的 HTML 标签（header、nav、main、article 等）
- **ARIA 属性**：为交互元素添加必要的 ARIA 标签
- **键盘导航**：支持 Tab、Enter、Space 等键盘操作
- **对比度**：确保文字和背景有足够对比度

## 🎬 动画
- **适度使用**：避免过度动画影响性能
- **时长合理**：快速响应（150-300ms），复杂过渡（300-500ms）
- **缓动自然**：使用合适的缓动函数
- **可关闭**：尊重用户的减少动画偏好

- [ ] 关键功能在小屏幕可访问
