# Axhub Make

一个专为 **产品经理** 和 **设计师** 打造的 AI 辅助开发工具，让你无需编程基础，也能为 Axure 原型创建自定义组件和页面。

## 📖 项目简介

本项目帮助你通过 AI 对话的方式，快速创建运行在 Axure 原型中的交互组件和页面。你只需要描述需求，AI 会自动生成代码，并帮助你调试和构建。

### 核心优势

- ✅ **零代码基础**：通过自然语言描述需求，AI 自动生成代码
- ✅ **快速迭代**：修改需求后，AI 自动更新代码
- ✅ **即插即用**：生成的 JS 文件可直接在 Axure 中使用
- ✅ **专业规范**：遵循最佳实践，代码质量有保障

### 可以创建什么？

- **元件（Elements）**：可复用的独立组件
  - 按钮、表单、图表、卡片等
  - 示例：`src/elements/button/`
  
- **页面（Pages）**：完整的业务页面
  - 用户列表、数据看板、设置页面等
  - 示例：`src/pages/sample-page/`

## 🚀 快速开始

### 第一步：启动项目

首次使用时，在项目目录下运行：

```bash
npm install
```

> 💡 **提示**：这一步只需要执行一次，会自动安装所有必需的工具。

安装完成后就可以启动项目了

```bash
npm run dev
```

### 第二步：与 AI 沟通需求

1. **打开 AI 助手**（如 Cascade、Cursor 等）
2. **描述你的需求**，AI 会引导你完善细节
3. **确认 spec 文档**，AI 会生成一份需求文档供你确认

**示例对话：**

```
你：我想做一个用户列表页面

AI：好的！请问这个页面需要哪些功能？
    1. 用户数据的展示方式（表格、卡片、列表）？
    2. 是否需要搜索和筛选功能？
    3. 是否需要分页？
    ...

你：用表格展示，需要搜索功能

AI：明白了。关于数据结构：
    1. 用户数据包含哪些字段？
    ...
```

### 第三步：AI 生成代码

AI 会根据你的需求自动生成：
- 组件代码（`index.tsx`）
- 需求文档（`spec.md`）

### 第四步：预览和调试

在浏览器中打开显示的地址（如 `http://localhost:51720`），你可以：
- 实时查看组件效果
- 修改配置项测试
- 查看需求文档

如果发现问题，直接告诉 AI："按钮颜色不对，改成蓝色"，AI 会自动修改代码。


## 第五步：导入 Axure

你可以通过以下两种方式将配置导入到 Axure 中：

---

### 方法一（推荐）

1. 在 **管理后台首页** 点击 **导出到 Axure**  
2、导出图片（默认包含代码）
3、打开 **Axure**，进入 **Axhub Runtime 元件**  
4、双击元件封面图片，替换下载的图片

---

### 方法二：使用项目链接一键更新

1. 在 **管理后台首页** 点击 **复制链接**  
2. 在 Axure 中选中 **Axhub Runtime 元件**  
3. 点击 **金手指 → AI 协作 → 一键更新**  
4. 将复制的 **链接** 粘贴进去并点击 **一键更新**  
5. 按金手指提示保存元件到 Axure 即可

## 💡 使用技巧

### 如何描述需求？

**好的描述：**
- ✅ "创建一个用户列表页面，用表格展示，包含姓名、邮箱、状态字段，支持搜索功能"
- ✅ "做一个按钮组件，有主要和次要两种样式，点击后触发事件"

**不够清晰的描述：**
- ❌ "做一个列表"（缺少细节）
- ❌ "做一个好看的按钮"（"好看"太主观）

### 如何修改需求？

直接告诉 AI 你想改什么：
- "把按钮颜色改成红色"
- "增加一个删除功能"
- "表格增加分页功能"

AI 会自动更新代码和文档。


## 🎯 典型工作流程

```
1. 描述需求 → AI 生成 spec.md
2. 确认需求 → AI 生成代码
3. 预览效果 → npm run dev
4. 发现问题 → 告诉 AI 修改
5. 构建生成代码 → npm run build
6. 使用生成代码 → 在 Axure 中配合 Axhub Runtime 元件库使用
```

## ⚙️ 技术说明

> 以下内容供技术人员参考，非技术用户可跳过

- **框架**：React + TypeScript
- **构建工具**：Vite
- **输出格式**：IIFE 格式的单文件 JS
- **兼容性**：编译为 ES5，兼容 Axure Runtime
- **React 引入**：通过 `window.React` 全局引入，无需打包
